# APEX-4 Morning - specifica tecnica iniziale

Stato: bozza operativa per motore/backtest. Nessun deploy.

## Obiettivo

Costruire una terza sezione indipendente dalla dashboard:

- OCTA: strategia azioni/fondi.
- Portafogli: tracker cifrato.
- APEX: rotazione settimanale 100% su un asset.

APEX deve produrre un segnale automatico e verificabile, con bottone manuale in app per rilanciare il motore.

## Versione timing

La richiesta aggiornata e': segnale il mercoledi mattina, usando dati freschi dopo apertura Xetra/Borsa Italiana.

Quindi la versione corretta non e' "close martedi" e non e' "close mercoledi sera". La bozza diventa:

- run automatica: mercoledi ore 09:45 Europe/Rome;
- dati live: ultimi prezzi intraday disponibili dopo apertura europea;
- esecuzione: appena letto il segnale, con attenzione a spread/liquidita;
- fallback: se il dato intraday non e' disponibile, usa ultimo segnale valido e mostra warning.

Motivo 09:45: alle 09:10 alcuni ETC/ETF possono avere ancora book/last price poco stabile o pochi scambi; 09:45 concede tempo al mercato di formare un primo prezzo utile. Possiamo testare anche 09:30.

## Regola base

Lookback: 8 settimane.

Osservazioni settimanali: mercoledi mattina, o giorno lavorativo precedente nella stessa settimana se mercoledi manca.

Ritorni:

- `ret_BTC = BTC[i] / BTC[i-8] - 1`
- `ret_GOLD = GOLD[i] / GOLD[i-8] - 1`
- `ret_SP500 = SP500[i] / SP500[i-8] - 1`

Priorita:

1. se BTC > 0 e BTC >= Oro -> BTC;
2. altrimenti se Oro > 0 e Oro > S&P 500 -> Oro;
3. altrimenti se S&P 500 > 0 e S&P 500 > Oro -> S&P 500;
4. altrimenti -> Cash/XEON.

S&P 500 non batte direttamente BTC se BTC supera Oro.

## Dati

Live:

- preferenza a strumenti quotati in EUR;
- intraday 5 minuti Yahoo chart per snapshot fresco;
- output audit con ticker, ora prezzo, currency, ultima barra.

Backtest:

- profilo `proxy`: BTC-USD, GC=F, ^GSPC convertiti in EUR via EURUSD=X, piu XEON.MI come cash quando disponibile;
- profilo `listed`: WBTC.PA, SGLN.MI, CSSPX.MI, XEON.MI, piu realistico ma con storia piu corta;
- test lookback 4/6/8/10/12/16 settimane;
- costi default 30 bps per cambio, poi sensitivity 10/30/60 bps.

## Da discutere dopo i primi numeri

- 09:30 vs 09:45;
- uso di `open` o ultima barra 5m per segnale live;
- 8 settimane puro vs buffer anti-whipsaw;
- quanto penalizzare i costi/spread;
- se usare sempre strumenti reali listati anche per backtest, accettando storia piu corta, o proxy lunghi per robustezza statistica.
