# Prove

Spazio separato per esperimenti, confronti e prototipi non ufficiali.

Regola per Codex e Claude:
- qui si possono mettere prove UI, backtest alternativi, strategie candidate e appunti tecnici;
- niente dentro `prove/` diventa automaticamente parte dell'app ufficiale;
- una prova entra in `dashboard/`, workflow o motori solo dopo decisione esplicita dell'utente;
- quando una prova viene promossa, aggiungere una nota in `HANDOFF.md`.

Questo evita confusione tra:
- versione ufficiale live;
- esperimenti temporanei;
- proposte fatte da assistenti diversi.
